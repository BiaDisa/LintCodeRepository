java虚拟机规范（JAVA SE8）

第二章 java虚拟机结构

所有在虚拟机规范之中没有明确描述的实现细节，都可以由设计者完全自主决定

2.1 class文件格式

     java编译后的二进制格式是*平台中立*的。该处的平台中立指的是不依赖于特定硬件及操作系统，实现方式是强制明确地定义了本来会跟平台相关的细节（例如字节序列），使得平台之间的实现都遵循一个模式，因此能够达到*平台中立*的效果。

2.2 数据类型

java虚拟机可操作数据类型分为2类：
1.原始类型（primitive type），亦译为：原生类型或基本类型
2.引用类型（reference type）

数值类型：
1.原始值
2.引用值
用于变量赋值、参数传递、方法返回、运算操作。

java虚拟机希望尽可能多的类型检查在程序运行之前完成

原始类型的值不需要通过特殊标记或别的额外识别手段来在运行期确定它们的实际数据类型，虚拟机的字节码指令本身就可以确定它的指令操作数的类型是什么。
