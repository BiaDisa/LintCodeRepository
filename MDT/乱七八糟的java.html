hashCode()

返回一个对象的哈希值。这个方法用于帮助哈希表更有效地执行。

1.每当同个对象在Java应用中被调用超过一次且在equals的比较中没有任何信息被修改，该方法就总会返回相同的整型值。当应用从一个执行跳转到另一个执行时，这个整数不需要保持相同。
2.如果两个对象在equals方法中相同，那么调用本方法时两个对象总会产生相同的整数结果。
如果两个对象在equals比较中是不同的，那么这个方法在两个对象上产生不同的整型值不是必须的。但必须注意，不同的对象产生不同的整型值能够提升hash表的性能。
This is typically implemented by converting the internal address of the object into an integer
这个方法的默认实现是将对象的内部地址转化成整数

Collections.sort(List)

  1. 根据特定的list中对象的comparable接口，对list进行排序。list中的对象都需要实现comparable这个接口，而且列表中的元素都需要可以互相比较（code e1.compareTo(e2)），不能因为compareTo方法，抛出类型转换异常（C lassCastException）。
    2.这个方法应当保证稳定性：相等的元素不能在排序中交换位置。
  2. 这个数组一定要是可修改的，但不应该是可变的。
    4.这个方法与list.sort这个使用特定数组且不使用的比较器方法不同。

抛出异常：
1.类型转换异常：如果list中包含无法比较对象时抛出。
2.不支持的操作异常：如果list的list迭代器（iterator）不支持set方法时抛出。
3.非法参数异常：（可选）如果实现类检测到list元素的自然排序违反了comparable的规定则会抛出。

比较规定：
1） x，y 的比较结果和 y，x 的比较结果相反。(自反性)
2） x>y，y>z，则 x>z。（传递性）
3） x=y，则 x，z 比较结果和 y，z 比较结果相同。（同一性）*

Collections.sort(List,(optionoal)Comparator)

详细同上，但是是使用输入的比较器对指定的list进行比较

binarySearch(list,key)

1.使用二分查找，在list中查找条件对应的对象。
2.list必须是按照比较器，升序排序好的。如果没有排序好，结果无法预知。如果根据key进行比较有多个符合结果的值，那么无法得知哪个结果会被返回。
3.该方法在实现了随机获取（random access，IMPL:RandomAccess）的list上运行时间为log（n）。如果没有实现该接口且list较大，该方法会使用基于迭代器的二分搜索，其实现为o(n)的遍历+o(logn)的元素比较。
4.返回值为查找key的索引（index）值，前提是key在list中，否则会返回一个key可插入位置（insertion point），**即按照升序排序，key该插入的位置，这个位置原来的值要么是第一个比key大的值，要么是数组的最后一位的后一个。**注意，当且仅当key出现在列表中后，返回值才保证>=0
5.异常：

类型转换异常（ClassCastException）：当数组中的元素无法相互比较，或者key和list中的元素无法全部相互比较（比较到了才会抛出）。

6.相关param：BINARYSEARCH_THRESHOLD   = 5000。如果大于该值且list未实现RandomAccess，则使用基于迭代器的二分查找（iteratorBinarySearch）;其余情况则使用索引二分查找（indexedBinarySearch）。

  1. 索引二分查找indexedBinarySearch（） ：直接使用list.get(),实现二分查找中关键节点的元素。
